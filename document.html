部署步骤

上传源码

将所有文件上传至服务器根目录，确保 PHP 版本 > 7.2，MySQL > 5.6。

运行安装程序

浏览器访问 http://你的域名/install.php，填写数据库信息并执行安装。

完成部署

安装成功，删除 install.php 后即可访问后台管理界面。

API 接口定义

所有客户端均通过此唯一接口与服务器通信，使用 POST 方法提交表单数据。

POST http://您的域名/Verifyfile/api.php
参数名 必填 说明
card_code 必填 用户输入的卡密（兼容参数名:card）
device_hash 必填 设备唯一标识（机器码），建议 MD5 后传入（兼容参数名: device）
app_key 可选 应用密钥，用于区分多应用或获取云端变量（后台"应用管理"中获取）

响应格式

服务端统一返回 JSON 格式数据，建议判断 code 字段处理结果。

200
验证通过
403
验证失败 / 已过期
429
请求过快（防刷）
Response Payload — application/json
{
  "code": 200,
  "msg": "OK",
  "data": {
    "expire_time": "2026-12-31 23:59:59",
    "variables": {             // 云变量（如已配置）
      "notice": "最新公告内容...",
      "download_url": "https://..."
    }
  }
}

import requests, hashlib

# ① 配置
api_url = "http://你的域名/Verifyfile/api.php"
card    = "VIP-88888888"
hwid    = hashlib.md5("my_computer_id".encode()).hexdigest()

# ② 请求
try:
    res = requests.post(api_url, data={
        "card_code": card,
        "device_hash": hwid,
        "app_key": ""   # 可选
    }, timeout=5).json()

    # ③ 处理结果if res['code'] == 200:
        print(f"✓ 验证通过，到期时间：{res['data']['expire_time']}")
    else:
        print(f"✗ 验证失败：{res['msg']}")

except Exception:
    print("网络连接失败")

云变量系统

在后台【应用列表 → 变量管理】中配置，验证成功后自动随响应下发。

私有变量
仅验证通过后下发

卡密验证通过后才返回给客户端。适用于加密 Key、VIP 直链、核心配置等敏感数据传输。

公开变量
仅需app_key

只需提交app_key 即可获取，无需消耗卡密次数。适用于公告、版本号、联系方式等公开数据。

云变量支持动态更新，后台修改后立即生效，客户端下次验证时自动获取最新内容，无需重新发版。

安全最佳实践

  • 安装后清理
    部署完成后立即删除 install.php，防止站点被恶意重置。
  • 修改默认密码
