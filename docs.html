环境要求与部署教程

GuYi Access 的服务端基于原生 PHP 开发，轻量且高效。您只需要一台基础的 Web 服务器即可完美运行。

  • PHP 版本： 建议 ≥ 7.2 (需开启 PDO 和 JSON 扩展)
  • 数据库： MySQL 5.6+

一键安装步骤：

将源码上传至网站根目录后，直接访问 http://您的域名/install.php，跟随图形化界面填写数据库账号密码，点击安装即可自动完成数据库表的创建。

通信协议与全局鉴权机制

为了保障极高的防破解能力，GuYi Access 采用 单点路由 (Single Endpoint) 设计。所有的请求均发送至同一个地址，通过 action 字段区分业务逻辑。

POST /Verifyfile/api.php
Content-Type: application/json

关于 AppKey 核心鉴权

每一个请求都必须在 JSON 体中携带 app_key 参数。这是您在后台创建应用时生成的 64 位十六进制字符串。它**既是应用的唯一标识，也是 AES 响应加密的 32 字节核心解密密钥**。

防逆向警告：请勿在客户端中明文拼接 AppKey！建议通过内存动态解密获取，或使用 VMP/天盾 等加壳工具保护您的主程序。

卡密验证接口 (核心)

用户首次输入卡密时调用。该接口会自动将卡密与用户的 device_hash 绑定，并激活计费周期。

请求参数

字段名类型必须说明
app_keyString必传后台获取的 64 位应用密钥
card_codeString必传用户输入的充值卡密
device_hashString可选机器码的 MD5 (推荐 CPU+主板组合算法)。不传则绑定 IP。
actionString可选默认为验证，也可显式传 verify

成功响应示例 (200 OK)

{
  "code": 200,
  "msg": "验证成功",
  "data": {
    "expire_time": "2026-12-31 23:59:59",
    "variables": { "Inject_DLL": "https://..." }
  }
}

心跳保活与防白嫖机制

为了防止用户验证成功后“拔网线”继续免费使用，强烈建议您在软件内加入心跳定时器（建议 3-5 分钟执行一次）。

请求参数同上，只需额外增加 "action": "heartbeat"。如果后台将卡密拉黑或卡密到期，接口将返回非 200 状态码，此时您的客户端应强制退出程序。

自助换绑设备

允许用户自行解除旧电脑的绑定，系统会自动扣除您在后台设置的“换绑惩罚时间（如 12 小时）”，大幅减少您的售后工作。

请求参数增加："action": "unbind" 以及 "new_device": "新机器码MD5"。

全局错误状态码大全

Code (状态码)含义说明客户端处理建议
200请求成功 / 验证通过正常放行
400请求参数缺失或格式错误检查 JSON 格式及必需字段
401卡密已到期弹窗提示用户续费后退出程序
403封禁、冻结或绑机冲突提示具体 msg，拦截进入
404卡密不存在或应用不存在提示卡密输入错误
429IP 请求频率超限 (防 CC)延迟重试，不应急于结束进程
500服务端配置异常联系服务端管理员
